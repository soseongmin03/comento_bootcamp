# 4주차 최종 프로젝트 - 회원가입과 To Do List

## 프로젝트 개요

이번 최종 프로젝트는 한 화면에서 `회원가입` 기능과 `To Do List` 기능을 사용할 수 있도록 구현한 정적 웹 프로그램입니다. 별도 서버나 DB 없이 브라우저 메모리의 배열에 데이터를 저장합니다.

- 실행 파일: `index.html`
- 스타일 파일: `style.css`
- 동작 파일: `script.js`
- 저장 방식: JavaScript 배열(`members`, `todos`)

## 3주차 피드백 반영 사항

- DOM 요소 조회 시 `requireElement()`로 필수 요소 존재 여부를 먼저 검증하도록 수정했습니다.
- `lastCheckedUserId`, `members`, `todos` 상태를 직접 수정하지 않고 `setLastCheckedUserId()`, `addMember()`, `addTodo()`, `removeTodoById()` 함수를 통해 변경하도록 정리했습니다.
- 이벤트 핸들러는 사용자 입력 처리와 검증 흐름을 담당하고, 화면 출력은 `renderMembers()`, `renderTodos()`가 담당하도록 역할을 분리했습니다.
- `renderTodos()`는 일정 id를 기준으로 삭제된 DOM만 제거하고 기존 항목은 재사용하도록 개선했습니다.
- 계산 기능이 없는 프로젝트이므로 `eval()`은 사용하지 않았고, 사용자 입력값은 DOM 텍스트로만 출력되도록 처리했습니다.

## 구현 기능

### 1. To Do List

- FR1: 일정 내용을 입력하고 `일정 추가` 버튼을 누르면 일정이 추가됩니다.
- FR2: 각 일정의 `삭제` 버튼을 누르면 해당 일정이 삭제됩니다.
- 추가 구현: 등록된 일정 개수와 등록 시간을 화면에 표시합니다.

### 2. 회원가입

- FR1: 회원가입 후 회원 정보가 `members` 배열에 저장됩니다.
- FR2: `중복확인` 버튼으로 아이디 중복 여부를 검사합니다.
- FR3: 비밀번호 입력 시 아래 규칙을 실시간으로 검사합니다.

비밀번호 규칙:

- 8자 이상
- 영문 포함
- 숫자 포함
- 특수문자 포함
- 공백 없음

## 화면 이벤트 설명

| 화면 요소 | 이벤트 | 동작 |
| --- | --- | --- |
| 아이디 입력창 | `input` | 아이디가 변경되면 기존 중복확인 결과를 초기화합니다. |
| 중복확인 버튼 | `click` | `members` 배열에서 같은 아이디가 있는지 검사합니다. |
| 비밀번호 입력창 | `input` | 비밀번호 규칙 충족 여부를 실시간으로 표시합니다. |
| 회원가입 폼 | `submit` | 필수값, 아이디 중복확인, 비밀번호 규칙을 통과하면 회원을 배열에 저장합니다. |
| 일정 추가 폼 | `submit` | 입력한 일정 내용을 `todos` 배열에 저장하고 목록을 다시 표시합니다. |
| 일정 삭제 버튼 | `click` | 선택한 일정 id를 기준으로 배열에서 제거하고 화면을 다시 표시합니다. |

## 주요 함수 설명

### 상태와 공통 함수

- `state`: 회원 배열, 일정 배열, 마지막으로 중복확인한 아이디를 관리하는 객체입니다.
- `requireElement(selector)`: 필수 DOM 요소가 없을 때 명확한 오류를 발생시켜 HTML 변경으로 인한 예외를 빠르게 확인합니다.
- `createId()`: `crypto.randomUUID()`로 회원과 일정에 사용할 고유 id를 생성합니다.
- `normalizeText(value)`: 입력값 앞뒤 공백을 제거해 빈 값 검사를 정확하게 합니다.
- `setLastCheckedUserId(userId)`: 마지막으로 중복확인한 아이디 상태를 변경합니다.
- `addMember(member)`: 회원 정보를 `members` 배열에 추가합니다.
- `addTodo(todo)`: 일정 정보를 `todos` 배열에 추가합니다.
- `removeTodoById(todoId)`: 선택한 id의 일정을 `todos` 배열에서 제거합니다.
- `setStatus(message, type)`: 화면 하단 알림 메시지를 변경하고 성공, 경고, 오류 스타일을 적용합니다.
- `setFieldMessage(element, message, type)`: 입력 필드 아래의 안내 메시지를 변경합니다.

### 회원가입 함수

- `isDuplicatedUserId(userId)`: `members` 배열에 같은 아이디가 있는지 확인합니다.
- `validatePassword(password)`: 비밀번호가 길이, 영문, 숫자, 특수문자, 공백 규칙을 충족하는지 객체로 반환합니다.
- `isPasswordValid(password)`: 모든 비밀번호 규칙을 통과했는지 Boolean 값으로 반환합니다.
- `updatePasswordRuleView(password)`: 비밀번호 규칙별 충족 여부를 화면에 반영합니다.
- `checkUserIdAvailability()`: 아이디가 비어 있는지, 중복인지 확인하고 사용할 수 있으면 `lastCheckedUserId`에 저장합니다.
- `renderMembers()`: `members` 배열을 기준으로 가입 회원 목록과 회원 수를 다시 그립니다.
- `resetSignupForm()`: 회원가입 완료 후 입력값과 검증 메시지를 초기 상태로 되돌립니다.
- `handleUserIdInput()`: 아이디 입력값이 바뀔 때 중복확인 상태를 초기화합니다.
- `handlePasswordInput()`: 비밀번호 입력 중 실시간 검사를 실행합니다.
- `handleSignupSubmit(event)`: 회원가입 제출 이벤트를 처리합니다. 조건을 모두 통과하면 회원 정보를 배열에 저장합니다.

### To Do List 함수

- `renderTodos()`: `todos` 배열을 기준으로 일정 목록과 일정 개수를 갱신합니다. id를 비교해 삭제된 항목만 제거하고 기존 DOM은 재사용합니다.
- `formatDateTime(date)`: 일정 등록 시간을 `YYYY-MM-DD HH:mm` 형식으로 변환합니다.
- `handleTodoSubmit(event)`: 일정 추가 제출 이벤트를 처리합니다. 빈 값이 아니면 일정 객체를 배열에 저장합니다.
- `handleTodoDelete(event)`: 클릭한 삭제 버튼의 일정 id를 찾아 해당 일정을 배열에서 제거합니다.

### 초기화 함수

- `bindEvents()`: 화면에서 발생하는 모든 이벤트 리스너를 등록합니다.
- `init()`: 초기 화면 렌더링, 비밀번호 규칙 초기화, 이벤트 연결을 실행합니다.

## 프로그램 동작 흐름

1. 페이지가 열리면 `init()` 함수가 실행됩니다.
2. 회원 목록과 일정 목록은 빈 배열 기준으로 렌더링됩니다.
3. 회원가입 시 아이디 중복확인과 비밀번호 규칙 검사를 먼저 수행합니다.
4. 검증을 통과하면 회원 정보가 `members` 배열에 추가되고 화면의 가입 회원 목록이 갱신됩니다.
5. 일정 추가 시 입력한 일정이 `todos` 배열에 추가되고 목록이 갱신됩니다.
6. 일정 삭제 시 선택한 일정의 id를 기준으로 배열에서 제거하고 목록을 다시 렌더링합니다.

## 구현하지 못한 부분

- DB 저장은 과제 조건에 따라 구현하지 않았습니다.
- 새로고침 후에도 데이터가 유지되는 기능은 구현하지 않았습니다. 현재 데이터는 브라우저 메모리에만 저장됩니다.
- 로그인 기능은 요구사항에 포함되지 않아 구현하지 않았습니다.
